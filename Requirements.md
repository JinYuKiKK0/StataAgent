# StataAgent 需求文档

## 数据源说明
本系统仅支持通过 CSMAR-PYTHON 接口获取数据，确保数据来源的标准化和高质量。CSMAR 提供经过预处理和标准化的金融经济数据，避免了用户上传数据可能带来的格式混乱和数据质量问题。

### 阶段一：需求解析与数据定义

这是整个链路的“蓝图”阶段，核心是建立概念到数据的映射。

- **变量池锁定：** 明确核心解释变量 (X)、被解释变量 (Y) 以及控制变量。
- **颗粒度与频率确认：** 确定所需数据的维度。例如，明确宏观变量（如市级数字金融指数）与微观变量（如银行个体财务数据）的频率（年度/季度），为后续合并做准备。
- **节点输出：** 生成一份结构化的《变量定义与数据需求清单》。

### 阶段二：理论预判与模型构建

在接触数据之前，必须先在底层构建逻辑闭环。

- **预期符号设定：** 根据经济学常识，预判核心解释变量对被解释变量的影响方向（正向促进或负向抑制），这将在后续作为 Agent 校验结果的“基准线”。
- **计量方程式确立：** 确定基准回归模型。例如设定包含个体与时间双向固定效应的面板数据模型：$Y_{i,t} = \alpha + \beta X_{i,t} + \gamma Controls_{i,t} + \mu_i + \delta_t + \epsilon_{i,t}$。
- **节点输出：** 实证设计大纲与底层数学逻辑。

### 阶段三：CSMAR数据获取与处理

通过 CSMAR-PYTHON 接口获取结构化的标准化数据，确保数据质量与一致性。

- **数据接口调用：** 使用 CSMAR-PYTHON 接口获取指定变量和时间范围内的面板数据。
- **主键对齐与数据合并：** 基于 CSMAR 标准化的标识符（如证券代码、年份），自动完成数据的合并与对齐。
- **数据标准化处理：** 对获取的原始数据进行格式转换和标准化，确保符合分析要求。
- **节点输出：** 标准化的待分析面板数据表 (.dta 或 .csv)。

### 阶段四：描述性统计与数据质量验证

对从 CSMAR 获取的高质量数据进行最终的质量确认，确保分析结果的可靠性。

- **极值与逻辑校验：** 运行基础统计命令，检查是否存在逻辑悖论（如资产负债率为负、比例变量超过 100%）。如果发现严重问题，Agent 立即中止并向用户抛出具体的错误报告。
- **缩尾处理 (Winsorization)：** 在确认数据主体无误后，对连续型变量进行上下 1% 的缩尾处理，剔除极端值对回归线的拉扯。
- **节点输出：** 描述性统计表格，以及确认无误的数据通行证。

### 阶段五：核心回归与动态反馈循环

这是验证阶段二预期的核心验证区。

- **基准回归执行：** 逐步加入控制变量和固定效应，观察核心系数 $\beta$ 的符号和显著性水平。
- **预期校验与动态调整：** 将跑出的结果与阶段二的预判进行比对。
- **机制触发：** 如果结果符合预期（符号一致且显著），则放行进入下一步的稳健性与异质性检验；如果结果不符合预期，Agent 触发调整机制（如尝试剔除具有严重多重共线性的控制变量，或调整固定效应维度），并重新运行；若多次调整依然失效，则中止并反馈。
- **节点输出：** 包含基准回归、稳健性检验和异质性分析的完整统计结果表格矩阵。
