# 研究工作流

本文档捕获从用户请求到实证结果的域工作流。它通过代理输入、输出和检查点来表达每个阶段，补充 `Requirements.md`。

## 不变量

- 数据源仅为 CSMAR。
- 最终分析数据集必须是单个分析粒度下的合并长表。
- 回归执行仅在数据质量检查点通过后才发生。
- 自动调整是有界的且可审计的；代理不得自由重写研究问题。

## 阶段概述

| 阶段 | 目标 | 必需输出 | 进入下一阶段的检查点 |
| --- | --- | --- | --- |
| S1 需求解析与数据定义 | 将主题和实证要求转化为结构化研究规范 | 变量定义清单和数据需求清单 | 核心变量、面板粒度和时间范围明确 |
| S2 理论预判与模型构建 | 定义预期符号、基准模型和实证逻辑 | 模型设计大纲和基准方程 | 基准回归设计和安全重试边界明确 |
| S3 数据清洗与拼接 | 获取 CSMAR 数据，标准化字段，合并源 | 一个标准化的长表 | 键、频率和连接一致 |
| S4 描述统计与质量拦截 | 在运行回归之前验证合并的面板 | 描述性统计和质量发现 | 没有阻塂数据问题 |
| S5 核心回归与反馈循环 | 运行回归，与期望比较，并决定是否允许有界重试 | 回归输出、稳健性输出、异质性输出和判断摘要 | 结果包完成或明确失败 |

## 阶段详情

### S1：需求解析和变量定义

- 输入：用户主题、实证步骤、实体提示、时间范围提示
- 代理工作：
  - 识别 `Y`、`X` 和控制变量候选
  - 确定目标分析粒度和数据频率
  - 定义所需的原始数据域
- 输出：
  - 变量定义表
  - 数据需求表
  - 初始研究规范

### S2：理论和模型设计

- 输入：来自 S1 的研究规范
- 代理工作：
  - 为核心关系分配预期符号或方向
  - 定义基准回归族
  - 选择候选固定效应和控制策略
- 输出：
  - 模型设计摘要
  - 基准方程
  - 允许的重试边界

### S3：CSMAR 获取、标准化和合并

- 输入：变量绑定和查询计划
- 代理工作：
  - 仅获取所需的 CSMAR 字段
  - 标准化 ID、日期、单位和缺失值编码
  - 将所有源合并到目标分析粒度
- 输出：
  - 一个合并的长面板
  - 获取日志
  - 谱系和覆盖摘要

### S4：描述性统计和质量检查点

- 输入：合并的长面板
- 代理工作：
  - 运行描述性统计
  - 检测不可能值、重复键、无效比率和大量缺失
  - 在适当时对连续变量进行缩尾处理
- 输出：
  - 描述性统计
  - 数据质量报告
  - 回归就绪的通过/失败决策

### S5：基准回归和反馈循环

- 输入：验证的数据集和基准模型计划
- 代理工作：
  - 从锁定的模型计划生成 Stata 代码
  - 通过 `stata-executor-mcp` 执行
  - 将系数符号和显著性与理论期望进行比较
  - 仅在批准的重试边界内运行有界重试
- 输出：
  - 基准回归表
  - 允许时的稳健性和异质性输出
  - 带有重试审计的最终判断摘要
