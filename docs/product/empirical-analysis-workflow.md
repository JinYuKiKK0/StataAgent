# 实证分析分阶段工作流

本文档是 StataAgent 产品流程的单一事实来源。它把需求约束与阶段状态机合并为同一份分阶段工作流定义，用于约束从研究请求到结果判断的完整实证分析链路。

## 不变量

- 数据源仅为 CSMAR。
- 最终分析数据集必须是单个分析粒度下的合并长表。
- 回归执行仅在数据质量检查点通过后才发生。
- 自动调整必须有界且可审计；Agent 不得自由重写研究问题。

相关工程约束见 `docs/engineering/agent-harness.md`，供应商接口限制见 `docs/references/CSMAR_PYTHON.md`。

## 运行状态机

```text
requested
-> specified
-> mapped
-> fetched
-> standardized
-> validated
-> modeled
-> executed
-> judged
-> completed | failed
```

附加约束：

- 每个阶段都必须输出显式契约对象或决策记录，不能只把结果留在提示上下文里。
- 只有当前阶段检查点通过后，工作流才能推进到下一状态。
- 如发生有界重试，必须在最终结果包中留下可审计记录，而不是静默回滚或覆盖。

## 阶段概览

| 阶段 | 目标 | 必需输出 | 进入下一阶段的检查点 |
| --- | --- | --- | --- |
| S1 需求解析与数据定义 | 将研究主题和实证要求转化为结构化研究规范 | 变量定义清单、数据需求清单、`ResearchSpec` | 核心变量、面板粒度和时间范围明确 |
| S2 理论预判与模型构建 | 定义预期符号、基准模型和实证逻辑 | 理论预期摘要、基准方程、模型计划 | 基准回归设计和安全重试边界明确 |
| S3 数据获取、标准化与拼接 | 获取 CSMAR 数据并转换为统一分析粒度 | `VariableBinding`、`QueryPlan`、标准化长表 | 键、频率和连接一致 |
| S4 描述统计与质量拦截 | 在运行回归前验证合并后的面板 | 描述统计、质量报告、`QualityDecision` | 没有阻断数据问题 |
| S5 核心回归与反馈循环 | 执行回归、对照理论并决定是否允许有界调整 | `StataRunPlan`、执行输出、判断摘要、`ResearchBundle` | 结果包完成或明确失败 |

## S1 需求解析与数据定义

### 目标

这是整个链路的蓝图阶段，核心是建立研究概念到数据需求的映射。

### 固定需求

- 锁定核心解释变量 `X`、被解释变量 `Y` 以及控制变量槽位。
- 确认目标分析粒度和数据频率，为后续拼接做准备。
- 明确所需数据域和最小变量定义集合。

### Agent 工作

- 识别 `Y`、`X` 和控制变量候选。
- 确定目标分析粒度和数据频率。
- 定义所需的原始数据域。

### 必需输出

- 变量定义清单
- 数据需求清单
- 初始 `ResearchSpec`

### 阶段检查点

- 核心变量不可为空。
- 时间范围和样本范围必须可审计。
- 粒度和频率必须足以驱动后续数据抓取与合并。

## S2 理论预判与模型构建

### 目标

在接触数据前先建立理论闭环，形成可验证的基准线。

### 固定需求

- 预判核心解释变量对被解释变量的影响方向。
- 确立基准回归族和最小模型设定。
- 锁定允许的调整边界，禁止无约束试错。

### Agent 工作

- 为核心关系分配预期符号或方向。
- 定义基准回归族。
- 选择候选固定效应和控制策略。

### 必需输出

- 理论预期摘要
- 基准方程
- 模型计划
- 允许的重试边界

### 阶段检查点

- 核心系数方向和经济学解释明确。
- 基准模型可被后续 Stata 模板直接消费。
- 调整边界被明确定义并可审计。

## S3 数据获取、标准化与拼接

### 目标

通过 CSMAR 获取所需数据，并将多源数据转换为可分析的统一长表。

### 固定需求

- 仅允许通过 CSMAR 接口获取数据。
- 主键、时间字段、单位和缺失值编码必须标准化。
- 合并结果必须回到单一目标分析粒度。

### Agent 工作

- 仅获取所需的 CSMAR 字段。
- 标准化 ID、日期、单位和缺失值编码。
- 将所有数据源合并到目标分析粒度。

### 必需输出

- `VariableBinding`
- `QueryPlan`
- 一个标准化的长面板
- 获取日志与来源摘要

### 阶段检查点

- 字段映射和查询计划可追踪。
- 键、频率和连接关系一致。
- 不可解析的合并必须阻断而不是静默回退。

## S4 描述统计与质量拦截

### 目标

在回归前执行最终的数据质量检查，并阻断严重问题。

### 固定需求

- 检查缺失、重复键、非法取值、极端值和逻辑悖论。
- 仅在主体数据通过检查后对连续变量执行缩尾。
- 回归前必须有明确的 pass/fail 决策。

### Agent 工作

- 运行描述性统计。
- 检测不可能值、重复键、无效比率和大量缺失。
- 在适当时对连续变量进行缩尾处理。

### 必需输出

- 描述性统计
- 数据质量报告
- 回归就绪的通过/失败决策

### 阶段检查点

- 严重数据问题会阻断流程并给出可诊断原因。
- 质量决策和缩尾摘要必须可被后续阶段引用。

## S5 核心回归与反馈循环

### 目标

执行基准回归并把结果与理论预期对照，按受控边界决定是否允许调整。

### 固定需求

- 从锁定的模型计划生成 Stata 代码。
- 逐步观察核心系数方向与显著性。
- 仅在已批准边界内运行有界调整，不得无限重试。

### Agent 工作

- 从锁定的模型计划生成 Stata 代码。
- 通过 `stata-executor-mcp` 执行。
- 将系数符号和显著性与理论期望进行比较。
- 仅在批准的重试边界内运行有界重试。

### 必需输出

- 基准回归表
- 允许时的稳健性和异质性输出
- 带有重试审计的最终判断摘要
- 最终 `ResearchBundle`

### 阶段检查点

- 成功路径下必须产出可审计结果包。
- 失败路径下必须明确说明失败原因和所尝试的有界调整。
